Boas práticas de Desenvolvimento Seguro e Organizado

1) Diretórios(Pastas):
	+ Sempre colocar o nome dos diretórios com a 1ª letra de cada palavra em maiúsculo, 
	sem espaços e caracteres especiais. Exemplo: "DocumentacaoOficial";

	+ O nome dos diretórios devem ser intuitivos, não coloque nomes confusos. Exemplo:"Teste Pru"; 

2) Arquivos:
	+ O assunto do commit deve ser claro dizendo o que a alteração está modificando e se limitar a 50 caracteres. 
	Exemplo: git commit -m 'Essa alteração corrige o front-end';

	+ Os nomes do arquivos devem ser claros, a primeira palavra estar sempre em letra minúscula, sem espaços e caracteres especiais. Exemplo: "docBoasPraticasGitHub.txt";

	+ Mantenha os arquivos dentro dos diretórios correspondentes;

3) Código:
	+ Seguir as convenções determinados pelo Yoshi durante a aula;
		- Métodos e variáveis começando com letra minúscula e as outras palavras com letra maiúscula (camelCase). Exemplo:"quebraLinha";
		- Classes com letras maiúscula (PascalCase). Exemplo: "VerificaConta";
		- O código inteiro deve ser autoexplicativo, com nomes decentes e que todos os integrantes entendam apenas lendo;
		...

	+ Manter todo o código com controle de acesso (encapsulamento);

	+ Separar adequadamente as classes e seus respectivos diretórios. Exemplo: diretório de teste contendo apenas testes dentro,
	diretórios de lógica apenas com classes lógicas, etc;
